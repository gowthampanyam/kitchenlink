# KitchenLink – Backend Service for Restaurant Management System

**KitchenLink** is the **backend service** powering a full-featured **Restaurant Management System**.  
It provides REST APIs, authentication, data models, and business logic for managing restaurant operations — including orders, menus, tables, payments, and staff roles.

The backend is built with **Django**, **Django REST Framework (DRF)**, and **PostgreSQL**.
It’s production-ready and deployable to **AWS** with Docker support.

---

## Key Features

- **Role-based access control** – Host, Waiter, Chef, Manager, Owner, and Maintenance Admin.
- **Complete restaurant flow** – from seating to orders, kitchen coordination, and payments.
- **Multi-role system** with permission elevation and approvals for restricted actions.
- **Menu management** with specials, pricing control, and audit logs.
- **Real-time kitchen/waiter updates** (via Django Channels or WebSockets).
- **Integrated payments** and **tip tracking**.
- **Reports** for sales, items, and earnings.
- **Modular architecture** for scalability and maintenance.
- **AWS-ready** deployment setup with Docker and CI/CD support.
